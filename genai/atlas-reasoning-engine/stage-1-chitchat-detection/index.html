Stage 1: Chit-Chat Detection

Overview‚Äã

The first stage of the Atlas Reasoning Engine determines whether the input is a valid query within scope or casual conversation (chit-chat).

Processing Time: 50-100ms
Type: Fast validation
Purpose: Filter out off-topic requests and casual greetings

What Happens in This Stage‚Äã

Input Analysis‚Äã

Atlas analyzes the incoming text to classify the intent:

Classification Categories‚Äã

CategoryDescriptionAtlas Response
‚úÖ Valid QueryBusiness-related question or requestContinue to Stage 2
‚ö†Ô∏è Chit-ChatGreeting, pleasantry, or social interactionSimple conversational response
‚ùå Out of ScopeRequest outside agent's capabilitiesPolite rejection with guidance

‚úÖ Valid Queries (Pass Through)‚Äã

These queries proceed to Stage 2:

"What's the status of order 12345?"
"Show me all open cases for Acme Corp"
"Create a new opportunity for John Smith"
"What are my tasks for today?"
"Find contacts in the San Francisco region"

‚ö†Ô∏è Chit-Chat (Simple Response)‚Äã

These get handled with simple responses:

"Hello" ‚Üí "Hi! How can I help you today?"

‚ùå Out of Scope (Rejected)‚Äã

These get politely rejected:

"What's the weather?" ‚Üí "I focus on Salesforce data. How can I help with your business?"
"Sing a song" ‚Üí "I'm a business assistant. What would you like to know about your data?"

How It Works‚Äã

1. Text Preprocessing‚Äã

Original Input: "Hey there! What's the status of my order?"

2. Intent Classification‚Äã

Atlas uses a classifier to determine the primary intent:

SignalWeightIndicates
Business termsHighValid query
Question wordsMediumInformation request
Greetings onlyHighChit-chat
Action verbsHighValid command
Social phrasesMediumChit-chat

3. Confidence Scoring‚Äã

# Pseudocode
intent_score = calculate_intent_confidence(tokens)

if intent_score > 0.8:
    return VALID_QUERY
elif intent_score > 0.3:
    return CHIT_CHAT
else:
    return OUT_OF_SCOPE

  • Token Savings: Avoids expensive LLM calls for simple greetings
  • Quick Response: Users get immediate feedback for casual interactions

Admins can tune the chit-chat detector:

Monitoring‚Äã

Performance Considerations‚Äã

Typical Chit-Chat Detection Performance:

  • Processing Time: 50-100ms per request
  • Chit-Chat Detection Rate: 10-20% of incoming queries
  • Out-of-Scope Detection Rate: 3-8% of queries
  • False Positive Rate: Less than 5% with well-tuned prompts
note

Monitor your agent's performance through Setup ‚Üí Einstein ‚Üí Einstein for Service. Salesforce provides built-in analytics for query classification and detection accuracy.

Troubleshooting‚Äã

Issue: Valid Queries Marked as Chit-Chat‚Äã

Symptoms:

  • Business queries get simple responses
  • Users report "agent doesn't understand"

Solutions:

  1. Lower confidence threshold (0.8 ‚Üí 0.7)
  2. Add business terms to allowlist
  3. Review false positive logs
  4. Retrain classifier with examples

Issue: Too Many Queries Pass Through‚Äã

Symptoms:

  • High processing costs
  • Many irrelevant queries in pipeline

Solutions:

  1. Raise confidence threshold (0.8 ‚Üí 0.9)
  2. Expand out-of-scope patterns
  3. Add domain-specific filters
  4. Review query patterns

Best Practices‚Äã

‚úÖ Do's‚Äã

  • ‚úÖ Allow greetings at conversation start
  • ‚úÖ Combine greeting + query in single input
  • ‚úÖ Monitor false positive/negative rates
  • ‚úÖ Customize responses for your brand
  • ‚úÖ Track chit-chat patterns over time

‚ùå Don'ts‚Äã

  • ‚ùå Set threshold too high (blocks valid queries)
  • ‚ùå Set threshold too low (wastes resources)
  • ‚ùå Ignore user feedback on misclassification
  • ‚ùå Use generic responses (poor UX)
  • ‚ùå Disable monitoring
